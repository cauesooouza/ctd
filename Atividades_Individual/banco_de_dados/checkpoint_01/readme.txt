Modelo logico de banco de dados para um "todo list"

foram criadas 4 entidades nas quais temos:
- user
- todo
- tasks
- task_history 

"USER" ira guardar cada usuario a utilizar o "todo list", e ira conter nome, email e senha

"TODO" servira para guardar cada a lista de tarefas crida pelo usuario, onde ira conter um nome e uma descrição

"TASKS" sera armazenadas as tarefas da lista de tarefas do usuario, nome da tarefa, descrição, data, ordem e estado

"TASK_HISTORY"  servira pra saber as estatisticas onde ira guardar as tarefas completas, arquivadas e que estão em processo.

Espero que seja simples a enteder o modelo, e útil.
